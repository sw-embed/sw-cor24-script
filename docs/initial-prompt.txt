== WHAT THIS PROJECT IS ==

sw-cor24-script implements "sws" (Software Wrighter Script), a Tcl-like
command language interpreter for a COR24 microprocessor OS. It provides
shell and text-editor scripting on COR24 FPGA hardware.

Key design decisions already locked in (from research.txt):
- Tcl-like command model: everything is a command, words separated by
  whitespace, { } blocks, " " quoted strings
- Language name: sws, file extension: .sws, binary: sws
- Written in C, compiled by tc24r (cross C compiler from sw-cor24-x-tinyc)
- Runs natively on COR24 hardware or emulator
- No AST initially — tokenizer → word list → command dispatch
- Variables via set/$ with $var.field record access
- Comparison via commands (eq, ne, lt, gt, le, ge), not operators
- Command substitution via (cmd args...) parentheses, not [brackets]
- Structured run result via $rc record (run-rc pragma)
- exists? for safe variable/field presence testing
- Result<Value, ScriptError> error model throughout

== CONSTRAINTS ==

- tc24r C subset: no structs, no malloc/free, no string library,
  no varargs, single translation unit, 24-bit int, 8-bit char
- All data structures must use parallel arrays or packed flat memory
- UART I/O only (character-level via memory-mapped addresses)
- The interpreter reads .sws scripts from UART or memory buffer
- Keep static allocation limits reasonable (e.g., 256 variables,
  64 nested blocks, 4KB line buffer)
